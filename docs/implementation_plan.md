# 한국형 간편결제(토스, 카카오페이) UI 도입 기획안

사용자님 말씀이 정확합니다! 방금 전의 모달은 글로벌 스탠다드(Stripe 방식)라서 한국 사용자들에게는 카드번호 16자리를 일일이 입력하는 것이 매우 귀찮게 느껴질 수 있습니다.

한국에서는 무조건 **토스페이, 카카오페이, 네이버페이** 같은 간편결제 버튼 하나로 끝내는 것이 국룰이자 전환율을 높이는 핵심입니다! 따라서 결제창(Mockup)을 한국형 최신 PG사(Toss Payments 등) 스타일로 완전히 탈바꿈하겠습니다.

## Proposed Changes

### [MODIFY] `Dashboard.jsx` (결제 모달 컴포넌트 교체)

글로벌 카드번호 입력창을 삭제하고, 아래 기능을 담은 **한국형 간편결제 위젯**으로 교체합니다.

1. **간편결제 선택 위젯 (Toss Payments 스타일)**
   - 최상단에 가장 많이 쓰는 3대장 버튼 배치: `토스페이(파란색)`, `카카오페이(노란색)`, `네이버페이(초록색)`
   - 하단에 일반 `신용/체크카드` 버튼 및 `계좌이체/무통장입금` 버튼 배치
2. **실제 결제 플로우 시뮬레이션**
   - 예를 들어 **"카카오페이"** 를 클릭하면 QR코드 화면이 모달 내부에 나타나고, "휴대폰에서 결제 완료하기" 버튼을 누르면 결제 성공으로 넘어가는 디테일한 한국형 플로우를 흉내냅니다.
3. **UX 개선**
   - 로고와 테마 색상을 맞추어 사용자에게 신뢰감을 주는 최신 한국형 커머스/SaaS 결제창 느낌을 100% 구현합니다.

## Verification Plan
- `Dashboard.jsx` 코드를 수정한 뒤 로컬 화면에서 결제 버튼을 누릅니다.
- 16자리 카드 입력창 대신 귀여운 간편결제 버튼(토스/카카오/네이버)들이 뜨는지 확인합니다.
- 특정 간편결제를 클릭했을 때 한국인에게 익숙한 결제 플로우(예: QR 인증 대기창)가 모의로 잘 넘어가는지 테스트합니다.
