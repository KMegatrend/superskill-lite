# 유지보수 및 콘텐츠 수정 가이드 (MAINTENANCE GUIDE)

본 문서는 **Antigravity Skills** 서비스의 화면(UI) 디자인이나 코드를 전혀 수정하지 않고도, **오직 텍스트 데이터만 안전하게 추가/수정**할 수 있도록 돕는 가이드라인입니다.

## 1. 텍스트 데이터 파일의 위치
사이트의 콘텐츠(문구)는 모두 아래 파일에 집중되어 있습니다.
👉 **경로**: `src/data/landingData.js`

이 파일만 수정하면 사이트 전체에 내용이 즉시 반영됩니다.

---

## 2. 자주 묻는 질문(FAQ) 추가 및 수정 방법

`landingData.js` 파일 안에는 `faqs` 라는 배열이 있습니다.
새로운 질문을 추가하고 싶다면, 배열 맨 끝에 새로운 객체를 쉼표(`,`)로 구분하여 추가하면 됩니다.

**수정 전 (예시):**
```javascript
export const faqs = [
  {
    id: 1,
    question: "질문 1",
    answer: "답변 1"
  }
];
```

**수정 후 (새로운 질문 추가):**
```javascript
export const faqs = [
  {
    id: 1,
    question: "질문 1",
    answer: "답변 1"
  },
  {
    id: 2, // 👈 고유한 ID(숫자)를 적어주세요
    question: "새로운 질문 내용",
    answer: "새로운 답변 내용"
  }
];
```

---

## 3. 스킬 팩(Showcase) 정보 수정 방법

동일한 파일의 `packFeatures` 배열에서 스킬 팩 내용을 수정할 수 있습니다.
각 팩은 다음과 같은 속성을 가집니다:

*   `badgeText`: 상단 배지에 표시될 텍스트 (예: '🎨 Designer Pack')
*   `title`: 큰 제목 (예: '전문가급 UI/UX 디자인')
*   `description`: 상세 설명 텍스트
*   `features`: 체크마크(✔️)와 함께 표시되는 핵심 기능 목록 (배열 형태)

원하는 팩(예: `wizard`)의 텍스트를 찾아 따옴표(`""` 또는 `''`) 안의 글자만 수정하시면 됩니다.

---

## 4. 변경 사항 확인 방법
파일을 수정하고 저장(`Ctrl + S`)하면, 로컬 개발 환경(`npm run dev` 실행 중인 상태)에서는 **새로고침 없이 실시간으로 화면에 반영**됩니다.
UI 코드는 절대 깨지지 않으니, 데이터 구조(중괄호, 대괄호, 따옴표)만 유지한 채 마음껏 텍스트를 수정하세요!
