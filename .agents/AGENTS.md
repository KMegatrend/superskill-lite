# Project Agents Rules

## 1. 사전 백업(Commit) 필수 의무화
작업 파일 유실 방지 및 안전한 복구를 위해, 코드를 크게 수정하거나 새로운 기능을 구현하기 **직전**에는 사용자가 명시적으로 요청하지 않더라도 **반드시 현재 상태를 Git으로 커밋(Commit)하여 백업을 남겨야 합니다.** 
- Commit 메시지는 "[Auto-Backup] Before starting [작업명]" 형태로 통일합니다.

## 2. 작업 완료 및 배포 프로세스 (Git Push 규칙)
작업이 완료되어 로컬 Git에 커밋이 되더라도, **절대 임의로 원격 저장소(GitHub)에 `git push`를 실행하지 마세요.**
1. 먼저 로컬 서버 환경에서 작업 결과물을 구동합니다.
2. 사용자에게 로컬 환경에서 직접 테스트 및 확인(Verification)을 요청합니다.
3. 사용자가 "로컬 확인 결과 이상 없음" 또는 "이제 올려줘" 라고 승인(Confirm)한 경우에만 `git push`를 수행하여 온라인 서버(Vercel 등)에 배포되도록 합니다.

## 3. 기기 변경(노트북 ↔ 회사PC) 시 자동 동기화 규칙
사용자가 다른 기기에서 작업하다가 넘어온 경우를 대비하여,화면 **새로운 대화 세션이 시작되거나 사용자가 "노트북으로 왔어", "작업 시작할게" 등으로 작업 재개를 알리면, 그 즉시 무조건 `git pull` 명령어를 먼저 실행하여 원격 저장소의 최신 코드를 다운로드(동기화)**해야 합니다.
- 충돌(Conflict)이 발생할 경우 즉시 사용자에게 알리고 해결 절차를 진행합니다.

## 4. 프로젝트 용어 및 화면 구분 규칙
프로젝트 관련 용어와 서비스 화면을 지칭할 때는 혼선을 방지하기 위해 다음 명칭을 일관되게 사용합니다.
- **프로젝트 명칭**: AI SuperSkill은 항상 **"슈퍼스킬"**이라고 부릅니다.
- **서비스 화면 구분**:
  1. **랜딩페이지**
  2. **회원페이지** (하위 메뉴: 마켓플레이스, 스킬 변환기, 스킬 번역기)
  3. **마이페이지**
